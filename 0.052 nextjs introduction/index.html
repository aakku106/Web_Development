🧭 Routing Comparison

React (Manual Routing)

// Install React Router
npm install react-router-dom

// App.js
import { BrowserRouter, Routes, Route } from 'react-router-dom';
import Home from './Home';
import About from './About';

function App() {
  return (
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
      </Routes>
    </BrowserRouter>
  );
}

❌ Challenges:

  • Manual setup required
  • Extra dependencies
  • Configuration needed

Next.js (File-Based Routing)

// No installation needed!
// Just create files in pages folder

  blog/

export default function Home() {
  return <h1>Home Page</h1>
}

// pages/about.js

  return <h1>About Page</h1>
}

✅ Benefits:

  • Zero configuration

🎨 Rendering Methods

React (Client-Side Rendering)

// React App
function App() {
  const [data, setData] = useState(null);
  
  useEffect(() => {
    // Fetch data after component mounts
    fetch('/api/data')
      .then(res => res.json())
      .then(setData);
  }, []);

  if (!data) return <div>Loading...</div>;
  
  return <div>{data.content}</div>;
}

How it works:

  1. Browser loads empty HTML
  2. JavaScript downloads
  3. React renders components
  4. Data fetched from API
  5. Content appears

Next.js (Multiple Options)

// Static Generation (SSG)
export async function getStaticProps() {
  const data = await fetch('https://api.example.com/data');
  return { props: { data } };
}

// Server-Side Rendering (SSR)
export async function getServerSideProps() {
  const data = await fetch('https://api.example.com/data');
  return { props: { data } };
}

// Component receives data as props
function Page({ data }) {
  return <div>{data.content}</div>;
}

How it works:

  1. Data fetched on server
  2. HTML generated with content
  3. Full page sent to browser
  4. Instant content display
  5. React hydrates for interactivity
