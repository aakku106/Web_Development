💻 SSR Implementation

Next.js (React) Implementation

pages/products/[id].js 
// Server-Side Rendering in Next.js
export async function getServerSideProps(context) {
  const { id } = context.params
  
  // This code runs on the server for EACH request
  try {
    const res = await fetch(`https://api.example.com/products/${id}`)
    const product = await res.json()
    
    return {
      props: {
        product,
        timestamp: new Date().toISOString()
      }
    }
  } catch (error) {
    return {
      notFound: true // Shows 404 page
    }
  }
}

export default function ProductPage({ product, timestamp }) {
  return (
    <div>
      <h1>{product.name}</h1>
      <p>Price: ${product.price}</p>
      <p>Stock: {product.stock} units</p>
      <small>Rendered at: {timestamp}</small>
    </div>
  )
}

Key Points:

  • ✅ getServerSideProps runs on every request
  • ✅ Perfect for dynamic data that changes frequently
  • ✅ Always returns fresh data
  • ✅ Can access request context (cookies, headers)

Nuxt.js (Vue) Implementation

pages/products/_id.vue 
<template>
  <div>
    <h1>{{ product.name }}</h1>
    <p>Price: ${{ product.price }}</p>
    <p>Stock: {{ product.stock }} units</p>
  </div>
</template>

<script>
export default {
  async asyncData({ params, $axios }) {
    // Runs on server for each request
    try {
      const product = await $axios.$get(
        `https://api.example.com/products/${params.id}`
      )
      return { product }
    } catch (error) {
      return { product: null }
    }
  }
}
</script>

Key Points:

  • ✅ asyncData fetches data before rendering
  • ✅ Automatic server-side rendering
  • ✅ Built-in error handling
  • ✅ Access to Nuxt context

SvelteKit (Svelte) Implementation

routes/products/[id]/+page.server.js 
// SvelteKit server-side load function
export async function load({ params, fetch }) {
  // Runs on server for each request
  try {
    const response = await fetch(
      `https://api.example.com/products/${params.id}`
    )
    const product = await response.json()
    
    return {
      product,
      timestamp: new Date().toISOString()
    }
  } catch (error) {
    return {
      status: 404,
      error: new Error('Product not found')
    }
  }
}
routes/products/[id]/+page.svelte 
<script>

</script>

<div>

  <p>Stock: {data.product.stock} units</p>
  <small>Rendered at: {data.timestamp}</small>
</div>

Key Points:

  • ✅ load function handles server logic
  • ✅ Clean separation of concerns
  • ✅ Automatic type inference
  • ✅ Built-in error handling

🎯 SSR Best Practices

✅ DO

  • Cache SSR responses when possible
  • Handle errors gracefully
  • Use TypeScript for type safety
  • Implement timeout for API calls
  • Monitor server performance

❌ DON'T

  • Access browser APIs in server code
  • Make unnecessary API calls
  • Ignore memory leaks
  • Skip error boundaries
  • Expose sensitive data
